1) create a local folder to store the repository folder ie: /home/Documents/repo/ 
2) clone the repository. the repository when cloned will look like this: /home/Documents/repo/CS116_May-Project
3) preform a pull action. This ensures that you have the correct and most recent files for the repo
4) use commits frequently and always remember to push all commits to origin so others can see your work.

TLDR: Make sure that you pull all requests before you start to get the most updated branch.
periodically commit changes when you complete a task. to make a commit you must add a commit message or comment explaining the change.
do this to help with debugging as reverting small changes is a lot easier than big changes.

When you are finished working remember to PUSH commits to a branch. if you do not nobody else besides you will be able to see you commits when they do a pull request.

I personally only like using the main branch for error free and bug free code. This ensures that there is always at least one functional program avalible for the project.
